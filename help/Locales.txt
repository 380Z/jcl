[Declaration]
property CharInfo[InfoType: Integer]: Char read GetCharInfo;
[ShortDescr]
Retrieves character typed locale information.
[Description]
The CharInfo property lets you read character typed locale information directly.
The InfoType is an Integer which specifies the locale information type which you
want to retrieve. You can specify any LOCALE_* information types as documented
in the Platform SDK documentation on GetLocaleInfo. Note that if the locale
information type requested is string based, only the first character of the
string is returned. This property should only be used for those information types
that are not directly supported by this class through it's properties.
[Quick Info]
Kind=Property
Visibility=Public

List of date formats that are available for this locale. The list is indexed by
a TJclLocaleDateFormats which is defined as:<br>
  TJclLocaleDateFormats = (ldShort, ldLong);<br>
As such the index determines whether the returned list contains the long or short
date formats. The returned string list is only valid as long as you don't read the
DateFormats property again with a different index. If you need this list to outlive
subsequent reads than assign it's contents to a different stringlist, for example: <br>
  <p class=syntax>
var
  ShortDateFormats: TStrings;
  LongDateFormats: TStrings;
  Info: TJclLocaleInfo;
  ...
  ShortDateFormats.Assign(Info.DateFormats[ldShort]);
  LongDateFormats.Assign(Info.DateFormats[ldLong]);
  </p>
