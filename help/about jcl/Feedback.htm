Feedback

General feedback

JCL is build by the community, but more importantly, for the community. Therefore we would like to hear from you. If you have any comments or suggestions we would appreciate it if you would drop us a note. There are several ways to get in contact with us.
  • You can always contact us directly by e-mail.
  • If you want to keep up to date about JCL then you can join the JCL mailing list either by going to JCL egroups page or by sending an email to egroups. You can also use this list to voice your opinion, comments or suggestions.

Bug reports

We have worked hard to eliminate as many bugs as possible and I like to think that we succeeded in finding them all. That, however, if is not realistic thinking. It is very likely that bugs are still present in the JCL. If you (think) you found one we would really like to know about it. You have several options for this, you can e-mail me or the Project JEDI testing coordinator, Matthias Thomas, directly. Alternatively you can also send a bugreport to the aforementioned mailing list. Never contact the author of the code directly. In any event, when you submit a bugreport please observe that the report should be detailed enough for us to (attempt to) reproduce it. Therefore it should contain at least the following information:
Field Description
Name Your name
E-mail Your e-mail address
Compiler The compiler you are using, for example Delphi 5, update pack 1
Operating System Operating system you are using, for example Windows NT 4, service pack 6
JCL Version of JCL. You can find the version in the JclBase unit. We need the value of the JclVersion constant. Alternatively you may supply the last modification date of the unit in which the code that the bug applies to resides.
Unit Name of the unit the code this bug-report applies to resides in.
Routine The name of the routine the bug report applies to, optionally including the sourcecode linenumber(s)
Description A description of the bug.
Steps to reproduce A detailed description of the steps to reproduce the bug.
Fix If possible a suggestion on how to fix the bug.
